> **Project:** ESP32-S3-CAM Rock-Paper-Scissors Game
> **Date:** 2026-03-04
> **Status:** 已批准

---

## 1. 项目概述

### 目标
基于 ESP32-S3-CAM 的剪刀石头布游戏，玩家通过手势与电脑对战，OLED 实时显示结果，蜂鸣器和 LED 提供声光反馈。

### 硬件配置
- **主控:** ESP32-S3-CAM (集成摄像头)
- **显示:** 0.96英寸 OLED (128x64, I2C)
- **输入:** 2个按键（开始、复位）
- **输出:** 无源蜂鸣器、三色LED

## 2. 系统架构

### 有限状态机

```
┌─────────┐    开始键    ┌────────────┐   倒计时结束   ┌─────────┐
│  IDLE   │ ──────────→ │ COUNTDOWN │ ────────────→ │ PLAYING │
│  待机   │              │   倒计时   │                │  识别中  │
└─────────┘              └────────────┘                └─────────┘
    ↑                                                  │
    │                    ┌────────────┐                  ↓
    │ 复位键            │  RESULT   │   2秒后自动      ┌─────────┐
    └───────────────── │   结果     │ ─────────────→ │ COUNTDOWN│
                       └────────────┘                 └─────────┘
```

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| IDLE | 待机，显示"等待开始" | 上电/复位完成 |
| COUNTDOWN | 3秒倒计时，显示"3-2-1" | 按下开始键 |
| PLAYING | 摄像头采集+手势识别 | 倒计时结束 |
| RESULT | 显示结果、声光提示 | 识别完成（置信度≥75%） |
| → COUNTDOWN | 自动进入下一局 | 结果显示2秒后 |

### 模块划分

| 模块 | 文件 | 功能 |
|------|------|------|
| StateMachine | `src/StateMachine.cpp/h` | 状态管理与状态转换 |
| HandRecognition | `src/HandRecognition.cpp/h` | 肤色分割+手指计数 |
| GameLogic | `src/GameLogic.cpp/h` | 随机手势、胜负判定、比分 |
| DisplayManager | `src/DisplayManager.cpp/h` | OLED显示封装 |
| AudioManager | `src/AudioManager.cpp/h` | 蜂鸣器音效 |
| LEDManager | `src/LEDManager.cpp/h` | LED控制 |

## 3. 核心算法

### 手势识别：肤色分割 + 手指计数

**流程：**
1. 采集摄像头图像 (320x240)
2. YCrCb 颜色空间肤色分割
3. 轮廓检测与二值化
4. 指尖检测与手指计数
5. 映射到手势：
   - 0-1手指 → ✊ 石头
   - 2手指 → ✌️ 剪刀
   - 5手指 → ✋ 布

**置信度计算：**
- 基于检测到的手指数量与轮廓面积
- 阈值：≥75% 为有效识别

## 4. 游戏逻辑

### 胜负判定（标准规则）
- ✊ 石头 > ✌️ 剪刀
- ✌️ 剪刀 > ✋ 布
- ✋ 布 > ✊ 石头

### 电脑策略
- 纯随机选择（各1/3概率）
- 使用 `random()` 函数

### 比分管理
- 玩家/电脑 分别计分
- 复位键清零比分

## 5. 输出模块

### OLED 显示内容

| 状态 | 显示内容 |
|------|----------|
| IDLE | "等待开始\n按开始键" |
| COUNTDOWN | "倒计时: 3/2/1" |
| PLAYING | "识别中..." |
| RESULT | "你:✊ 电脑:✌️\n你赢了! 3:2" |

### 蜂鸣器音效

| 场景 | 频率 | 时长 | 间隔 |
|------|------|------|------|
| 倒计时 | 1000Hz | 100ms | 900ms |
| 玩家获胜 | 523Hz→587Hz→523Hz | 500ms | - |
| 电脑获胜 | 587Hz→523Hz | 400ms | - |
| 平局 | 523Hz+659Hz | 300ms | - |

### LED 指示

| 结果 | 颜色 | 时长 |
|------|------|------|
| 玩家赢 | 绿色 | 2秒 |
| 电脑赢 | 红色 | 2秒 |
| 平局 | 黄色 | 2秒 |

## 6. 硬件接线

### ESP32-S3-CAM 引脚
| 功能 | GPIO |
|------|------|
| OLED SDA | GPIO 4 |
| OLED SCL | GPIO 15 |
| 按键开始 | GPIO 0 |
| 按键复位 | GPIO 47 |
| 蜂鸣器 | GPIO 21 |
| LED 红 | GPIO 48 |
| LED 绿 | GPIO 45 |
| LED 蓝 | GPIO 21 (复用蜂鸣器) |

### 硬件注意事项
- 摄像头使用自带CSI接口
- OLED 使用 I2C (地址 0x3C)
- LED 采用共阴极接法
- 蜂鸣器需NPN三极管驱动

## 8. 后续优化方向

1. **手势识别升级**: 如75%阈值效果不佳，可升级为TFLite模型
2. **OLED优化**: 考虑SSD1309 (128x128) 获得更大显示空间
3. **功耗优化**: 关闭摄像头省电模式
4. **扩展功能**: 蓝牙对战、积分排行榜等
